Vision/problem statement: 

Apt (Advanced Packaging Tool) is a very popular package management tool in the Debian Linux community. One can update one's distribution simply by saying "apt-get dist-upgrade" or one can get any package installed simply by saying "apt-get install package-name". Several efforts underway are:

  • apt-rpm: to use similar apt-get functionality in red-hat distro
  • ipkg : to have a apt-get-ish functionality for familiar linux distro for ipaqs .

So the aim of Apt-get for spkg would be to modify the apt-get tool for reuse and try it for a strong-arm based handheld - Simputer , which runs a spkg packaging system.

Scope and initial approach/plan: 

  • Analysis of domain dependancy - particularly the difference between a full-fledged PC linux distro and a handheld distro.
  • Fishing out and modularizing the re-usable part. Typically the syncing and mirror finding algorithms.
  • Making a template for the individual ideosycracies of distros which can reuse the tool hence-forth.
